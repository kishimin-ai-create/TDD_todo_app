Add ES target/lib in backend tsconfig**

`backend/tsconfig.json` で `target`/`lib`
を落としているため TypeScript 既定（ES5）で解釈され、`backend/src/index.test.ts`
の `async` テストで `Promise` が解決できず `npm run typecheck` が `TS2468`
で失敗します。`check` ジョブ（`.github/workflows/backend.yaml`）は毎回
`npm run typecheck` を実行するので、このままだとCIが安定して通りません。

まわりの型解決ができる前提に揃えています。
