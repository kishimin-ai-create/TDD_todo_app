# 2026-04-22 作業日誌

## 22:38 作業記録

## 今日取り組んだこと

今日は、GitHub Copilot CLI をこのリポジトリ運用に馴染ませるための `.github/` 整備を中心に進めた。単に custom agent を増やすのではなく、**どの情報を参照し、どのファイルは触らず、どの定型作業をどの agent に任せるか** をルールとして固定することを目的にした。

## 背景

このリポジトリでは、実装そのもの以外にも次のような定型作業が多い。

- 実装内容を記事にする
- backend API 変更後に OpenAPI を更新する
- レビューコメントを見て、修正か返信かを判断する
- ここまでの作業を要約する

これらを毎回会話ベースで指示すると揺れやすいため、Copilot が最初から参照できる `.github/` の構造とルールを整えた。

## 変更したこと

### 1. custom agent と prompt の整備

次の agent / prompt を追加または調整した。

- `ArticleWriterAgent`
- `OpenApiWriterAgent`
- `WorkSummaryAgent`
- `ReviewResponseAgent`
- `write-article.prompt.md`
- `summarize-work.prompt.md`

特に `WorkSummaryAgent` は、当初想定していた prompt ベースの `/summarize-work` が Copilot CLI ではそのまま使えなかったため、CLI で確実に呼べる `@WorkSummaryAgent` 前提へ切り替えた。

関連ファイル:

- `.github/agents/ArticleWriterAgent.agent.md`

- `.github/prompts/summarize-work.prompt.md`

### 2. 共通 instructions の整理

`.github/instructions/copilot-instructions.md` に、実装時の前提になるルールを追加した。

- 保護対象ファイルは勝手に編集しない
- 設計書は `docs/design/` を一次参照先にする
- レビューコメントは `review/` を一次参照先にする
- backend API 開発後は OpenAPI を更新する

これによって、設計意図やレビュー文脈を無視してコードだけから判断する場面を減らせる形になった。

関連ファイル:

- `.github/instructions/copilot-instructions.md`
- `.github/rules/protected-paths.md`

### 3. `.github/README.md` の整備

使える custom command / custom agent の一覧と呼び出し例を `.github/README.md` にまとめた。さらに、新しいコマンドを追加したときは、この README へ同じタスク内で使い方を追記するルールも入れた。

関連ファイル:

- `.github/README.md`

## 気づき

- Copilot の custom prompt と custom agent は似て見えるが、Copilot CLI では同じようには扱えない
- リポジトリに合わせた運用を作るには、agent 追加だけでなく instructions と README の整備がセットで必要
- レビュー対応のような半自動化したい作業は、入力元ディレクトリを明確に決めておくと扱いやすい

## 保留・次に気をつけたいこと

- `WorkSummaryAgent` は diary へ追記する運用に切り替えたので、今後は同日の複数回実行時に記録が自然につながるかを見たい
- custom agent が増えてきたので、役割の重複が出てきたら README と instructions を再整理したい

今日は、GitHub Copilot CLI をこのリポジトリ専用の作業基盤として使いやすくするための整備を進めた。ポイントは、**agent を増やすこと自体ではなく、参照先・出力先・更新ルールまで含めて `.github/` に載せること** だった。
