## クロスプラットフォーム対応 — `test:report` スクリプトのパス区切り文字修正

### 背景・目的

GitHub Actions（Linux CI）で `npm run test:report` が失敗していた。  
エラーメッセージは以下のとおりで、スクリプトへのパスが正しく解決されていなかった。

```
Error: Cannot find module '/home/runner/work/TDD_todo_app/TDD_todo_app/frontend/..scriptsmerge-vitest-reports.cjs'
```

### 根本原因

`frontend/package.json` の `test:report` スクリプトに Windows スタイルのバックスラッシュ（`..\\scripts\\`）が使われていた。  
Linux シェルでは `\` がエスケープ文字として解釈されるため、パス区切り文字として機能せず、  
`..scriptsmerge-vitest-reports.cjs` という存在しないフラットなファイル名に化けていた。

### 修正内容

`frontend/package.json` の `test:report` スクリプトを 1 行修正した。

| 変更前 | 変更後 |
|---|---|
| `node ..\\scripts\\merge-vitest-reports.cjs ...` | `node ../scripts/merge-vitest-reports.cjs ...` |

フォワードスラッシュ（`/`）は Windows・Linux・macOS のすべてで有効なパス区切り文字であるため、この変更により CI 環境での互換性が確保される。

### 変更ファイル

| ファイル | 変更内容 |
|---|---|
| `frontend/package.json` | `test:report` スクリプトのパスを `..\\scripts\\` → `../scripts/` に変更（1行） |

### 検証結果

| チェック | 結果 |
|---|---|
| `npm run typecheck` | ✅ exit 0 |
| `npm run lint` | ✅ exit 0 |

コミット: `1fff907` — `fix: use forward slash path in test:report script for Linux compatibility`（push 済み）

### 未完了・備考

特になし。修正は 1 行のみで影響範囲も限定的。typecheck・lint ともにグリーン確認済み。
