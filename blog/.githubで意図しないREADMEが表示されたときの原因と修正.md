# .githubで意図しないREADMEが表示されたときの原因と修正

## 対象読者

- GitHub 上のディレクトリ表示で README の見え方に困った人
- `.github/` 配下に運用ドキュメントを置いている人
- ファイル内容ではなく、配置や命名が原因の問題を切り分けたい人

## エラー概要

GitHub 上で `.github` ディレクトリを開いたときに、表示したかったルートの `README.md` ではなく、`.github/README.md` の内容が表示されていた。

今回のケースでは、`.github/README.md` に custom command の使い方を書いていたため、`.github` 配下の説明用ドキュメントとしては便利だった。一方で、GitHub の表示上はそのファイルが `.github` ディレクトリの README として扱われ、意図しない見え方になっていた。

問題のポイントは、**README の中身ではなくファイル名が `README.md` だったこと** だった。

## 原因

原因は GitHub のディレクトリ表示仕様にあった。

`.github` 配下に `README.md` が存在すると、GitHub はそのファイルをそのディレクトリの README として自動表示する。  
そのため、ルート README を見せたいつもりでも、`.github` ディレクトリを開いたときには `.github/README.md` の方が優先的に表示される。

今回の構成では、次の状態になっていた。

- ルートの README: `README.md`
- `.github` 配下の運用ドキュメント: `.github/README.md`

この構成だと GitHub の動作としては自然だが、見せたいものと表示されるものがずれていた。

## 結論

修正はシンプルで、`.github/README.md` を README ではない名前へ変更した。

- 変更前: `.github/README.md`
- 変更後: `.github/CUSTOM_COMMANDS.md`

あわせて、このファイルを参照していた記述も追従修正した。

関連ファイル:

- `.github/CUSTOM_COMMANDS.md`
- `blog/GitHub Copilot CLI向けにリポジトリ専用エージェントと運用ルールを整備した.md`
- `diary/20260422.md`

この変更によって、`.github` 配下には `README.md` が存在しなくなり、GitHub 上で `.github` ディレクトリを見たときに意図しない README 表示が起きない構成にできた。

## まとめ

今回の問題は、ドキュメントの中身が悪かったのではなく、**GitHub が `README.md` という名前を特別扱いしていること** が原因だった。

この件で意識しておきたいのは次の 2 点。

- GitHub 上の見え方は、ファイル内容だけでなくファイル名にも強く依存する
- `.github/` のような運用ディレクトリに説明ファイルを置くときは、`README.md` にするかどうかを先に考えた方がよい

運用ドキュメントを増やしていく場合でも、GitHub の自動表示対象になる名前を避けるだけで、余計な混乱をかなり減らせると分かった。
